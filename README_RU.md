# LuCI Mobile — русская версия

Это форк приложения [LuCI Mobile](https://github.com/cogwheel0/luci-mobile)
(управление роутерами OpenWrt) с переведённым на русский язык интерфейсом.
Переведены все видимые пользователю надписи: кнопки, заголовки, подсказки,
сообщения об ошибках и диалоги. Внутренние идентификаторы (вызовы RPC, ключи,
логи) намеренно оставлены без изменений, чтобы не сломать работу приложения.

Лицензия исходного проекта — GPLv3.

## Как получить готовый APK

Собрать APK «вживую» в чате невозможно, поэтому есть два пути.

### Способ 1 — без компьютера, через GitHub Actions (рекомендуется)

1. Создайте аккаунт на https://github.com (если его нет).
2. Создайте новый **пустой** репозиторий, например `luci-mobile-ru`.
3. Загрузите туда содержимое этого архива (кнопка *Add file → Upload files*,
   либо через `git push`). Важно: в архиве есть папка `.github/workflows` —
   она должна сохраниться.
4. Откройте вкладку **Actions** в своём репозитории. Найдите workflow
   **«Build RU APK»** и нажмите **Run workflow** (или он запустится сам после
   загрузки файлов).
5. Дождитесь окончания сборки (≈10–15 минут, статус — зелёная галочка).
6. Откройте завершившийся запуск → внизу в разделе **Artifacts** скачайте
   `luci-mobile-ru-apk`. Внутри — файл `app-release.apk`.
7. Перенесите APK на телефон и установите (потребуется разрешить установку
   из неизвестных источников в настройках Android).

> APK подписывается одноразовым ключом, который создаётся прямо во время сборки,
> — секреты репозитория не нужны. Минус: при каждой новой сборке ключ другой,
> поэтому, чтобы обновить приложение, сначала удалите предыдущую версию.

### Способ 2 — собрать локально (нужен установленный Flutter)

Требуется [Flutter SDK](https://docs.flutter.dev/get-started/install)
(версия 3.35.x) и Android-toolchain.

```bash
flutter pub get

# Быстрый устанавливаемый сборочный вариант (debug-подпись):
flutter build apk --debug
# готовый файл: build/app/outputs/flutter-apk/app-debug.apk

# Либо оптимизированный release (создайте свой keystore и android/key.properties
# по примеру из .github/workflows/build-ru-apk.yml):
flutter build apk --release
# готовый файл: build/app/outputs/flutter-apk/app-release.apk
```

Установка на подключённый по USB телефон: `flutter install`.

## Что было изменено

- Переведены строки интерфейса в файлах `lib/screens/**`, `lib/widgets/**`,
  `lib/state/app_state.dart`, `lib/utils/http_client_manager.dart`,
  `lib/utils/url_parser.dart`, `lib/models/client.dart`, `lib/models/interface.dart`,
  `lib/config/app_config.dart`, `lib/main.dart`.
- Сохранены без перевода: имена методов/полей RPC, ключи хранилища, лог-сообщения,
  тестовые (mock) данные, ключевое слово `REVIEWER` (сверяется с вводом),
  значение `N/A` (используется в сравнении), стандартные термины (`Wi-Fi`, `SSID`,
  `DNS`, `OpenVPN`, `WireGuard`, `OpenWrt`).
- Добавлен workflow `.github/workflows/build-ru-apk.yml` для сборки APK.

## Важное замечание о проверке

Перевод выполнен заменой строковых литералов целиком (с сохранением кавычек),
структура кода не менялась: число изменённых строк совпадает один-в-один.
Тем не менее финальная компиляция не запускалась — обязательно проверьте сборку
(`flutter analyze` и саму сборку APK) перед использованием.
